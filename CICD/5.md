## Git

- Git considers files not directories

- restore comes in two forms
  - staged: to move the change from staging area to working tree
  - normal: to clear the changes from working tree

- Adding only modified files

```bash
git add -u
```

- Adding all changes to staging area

```bash
git add -A
```

- Adding only changes w.r.t current directory

```bash
git add .
```

- We want git to explicitly ignore some files or folder, we use `.gitignore`. [Refer Here](https://www.atlassian.com/git/tutorials/saving-changes/gitignore)
- Generally for all the popular language combinations we can generate a [.gitignore file](https://www.toptal.com/developers/gitignore)

## Git contd

### Branches in Git

- Branches allow parallel development. we might need paralled development for
  * multiple customers
  * muliple versions
  * poc (proof of concept)
- HEAD pointer points to Branch which points to latest commit on that branch
- In git the default branch is master
- Branches & Tags are considered as reference objects
- [Refer Here](https://www.atlassian.com/git/tutorials/using-branches) for git branch docs from attlasian
- When we want changes from one branch to another we have the following options
  * merge
  * rebase
  * cherrypick
- basic merge process
  * checkout to the destination branch where the merge has to happen
  * execute the command to merge from source

#### Practical branching

- initalize a new repo
- create a commit
- now view the branches

```bash
git branch
```

- Rename the branch from master to main

```bash
git branch -m main
```

- Add few more commits
- Now we will be creating a new branch `rel_v1.0`

```bash
git branch rel_v1.0
```

- And create a commit C3

