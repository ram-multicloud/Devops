
## Let's continue commits and create a mental model (visual git)

- Let's create directory and make commits

```bash
mkdir commit_graphs
cd commit_graphs/
git init
touch Readme.md
git status
git add .
git commit -m "first"
```

- Now let's make some more changes

```bash
echo "hello" > Readme.md
git status
git add .
git commit -m "second"
```

- Few more

```bash
echo "hello1" >> Readme.md
git status
git add .
git commit -m "third"

echo "hello2" >> Readme.md
git status
git add .
git commit -m "fourth"

echo "hello3" >> Readme.md
git status
git add .
git commit -m "fifth"
```

- Git always points to the latest commit.
- What git is pointing to is denoted by `HEAD`.
- In git, this graph starts with a default name called `master`.

- Commit id is a SHA1 Hash of:
  - previous commit
  - current changes
  - author
  - datetime
  - message

- Git only tracks files, not folders.

- Using `git add`

```bash
git add <filename>
git add .   # Add all changes in current and child directories
git add -A  # Add all changes
git add -u  # Add only modified changes
```

- Let's learn what is untracked and modified:
  - **untracked**: a new file which was not part of git commits
  - **modified**: existing file which was part of git commits, changed
  - **deleted**: existing file which was part of git commits, deleted
  - **renamed**: (it will appear in the staging area)

- git log --oneline --graph
