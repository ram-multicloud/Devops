# Types of Variables in GitHub Actions

In GitHub Actions, variables come in a few distinct types based on **scope** and **purpose**.

## 1. Environment Variables

Standard key-value pairs available to steps/jobs, similar to shell env vars.

- **Default env vars**: Provided by GitHub (e.g., `GITHUB_REPOSITORY`, `GITHUB_SHA`, `GITHUB_ACTOR`, `RUNNER_OS`)
- **Custom env vars**: Defined by you at workflow, job, or step level using `env:`

```yaml
env:
  NODE_ENV: production

jobs:
  build:
    env:
      REGION: us-east-1
    steps:
      - run: echo "Deploying to $REGION"
        env:
          DEBUG: true
```

## 2. GitHub Context Variables

Accessed via `${{ github.* }}` — metadata about the workflow run itself.

Examples: `github.repository`, `github.ref`, `github.sha`, `github.actor`, `github.event_name`

## 3. Secrets

Encrypted variables for sensitive data (tokens, passwords, keys), accessed via `${{ secrets.* }}`.

- Repository secrets
- Environment secrets
- Organization secrets

```yaml
steps:
  - run: echo "${{ secrets.API_KEY }}"
```

## 4. Configuration Variables (vars context)

Non-sensitive, reusable values — the "public" counterpart to secrets. Accessed via `${{ vars.* }}`.

- Repository variables
- Environment variables (vars)
- Organization variables

```yaml
steps:
  - run: echo "Deploying to ${{ vars.DEPLOY_ENV }}"
```

## 5. Job Outputs / Step Outputs

Values passed between steps or jobs, accessed via `steps.<id>.outputs.*` or `needs.<job>.outputs.*`.

```yaml
- id: set-version
  run: echo "version=1.2.3" >> "$GITHUB_OUTPUT"

- run: echo "Version is ${{ steps.set-version.outputs.version }}"
```

## 6. Matrix Variables

Defined in a build matrix strategy, accessed via `${{ matrix.* }}`.

```yaml
strategy:
  matrix:
    node: [16, 18, 20]
steps:
  - run: echo "Testing on Node ${{ matrix.node }}"
```

## 7. Input Variables (workflow_dispatch / reusable workflows)

User-supplied inputs, accessed via `${{ inputs.* }}` or `github.event.inputs.*`.

```yaml
on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Deploy target'
        required: true
        type: string
steps:
  - run: echo "Deploying to ${{ inputs.environment }}"
```

## 8. Default Environment Variables (Runner-provided)

Provided automatically by the runner (`RUNNER_OS`, `RUNNER_TEMP`, `RUNNER_ARCH`, etc.), not the `github` context.

---

## Quick Scope Summary

| Type | Context prefix | Sensitive? |
|---|---|---|
| Env vars | `env.*` / plain shell | No |
| Github context | `github.*` | No |
| Secrets | `secrets.*` | Yes |
| Config vars | `vars.*` | No |
| Outputs | `steps.*` / `needs.*` | Depends |
| Matrix | `matrix.*` | No |
| Inputs | `inputs.*` | Depends |
