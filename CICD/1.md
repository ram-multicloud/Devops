## CI/CD

**CI/CD** stands for **Continuous Integration / Continuous Delivery (or Deployment)**. It is a set of practices and tools that automate the process of integrating code changes, testing them, and delivering software reliably and frequently.

| Term | Full Form | Core Idea |
|------|-----------|-----------|
| CI | Continuous Integration | Merge and test code changes frequently and automatically |
| CD | Continuous Delivery | Ensure every change is releasable at any point |
| CD | Continuous Deployment | Automatically release every passing change to production |

## The Problem — Life Before CI/CD

To understand the *why* of CI/CD, consider this real-world scenario.

### Company: SuperKart Pvt Ltd

> An online grocery store planning a major release before Diwali.

|------|-----------|----------------|

## Big Bang Integration — Why It Fails

This approach — where all components are built independently and integrated only at the end — is called **Big Bang Integration**. It is considered the **worst possible release strategy**.

### Problems with Big Bang Integration

```
┌──────────────┐     ┌───────────────┐     ┌─────────────────┐
│   Backend    │     │   Frontend    │     │     Testing      │
│  (6 months) │  +  │  (6 months)  │  →  │  (post Aug 31)  │
└──────────────┘     └───────────────┘     └─────────────────┘
                                                    │
                                    ┌───────────────▼──────────────────┐
                                    │ Hundreds of defects surface at   │
                                    │ once, close to release deadline  │
                                    └──────────────────────────────────┘
```

| Issue | Impact |
|-------|--------|
| Integration happens too late | Defects found near deadline — no time to fix |
| Components developed in isolation | Assumptions may not match across teams |
| High cost of bug fixing | Bugs found late are 10–100× more expensive to fix |
| Release delays | Diwali launch missed due to last-minute fire-fighting |
| Team blame games | Each team blames the other — morale drops |

> **Root cause:** Integration is an event, not a process. In Big Bang, it's a single terrifying event right before release.

## The Solution — Continuous Integration (CI)

CI flips the model: **integration is continuous**, not deferred.

### Revised Plan with CI

```
Timeline (2026)
│
├── Feb 26     → Ravi delivers BASIC backend (not complete — just enough to integrate)
├── Feb 26     → Phani delivers BASIC frontend
├── Feb 26     → Imran's team starts testing immediately
│
└── From Feb 26 onwards:
       For EVERY code change pushed by any team member:
         → Code is automatically integrated
         → Tests are automatically run
         → Defects are surfaced immediately
         → Teams fix defects quickly (context still fresh)
```

### CI Integration Flow

```
Developer Pushes Code
        │
        ▼
┌───────────────────┐
│   Version Control │  (Git / GitHub / GitLab)
│   (Trigger CI)   │
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│   Build Server    │  (Jenkins / GitHub Actions / GitLab CI)
│   - Pull code     │
│   - Compile/Build │
│   - Run Tests     │
└────────┬──────────┘
         │
    ┌────┴────┐
    │         │
  PASS       FAIL
    │         │
    ▼         ▼
Deploy     Notify Developer
to next    (Fix immediately —
stage      context is fresh)
```

## Key Principles of CI

### 1. Integrate Early and Often

Don't wait weeks or months. Push small changes. Merge frequently. Every commit triggers integration.

### 2. Automate Everything

Manual builds and tests are unreliable and slow. CI requires automation:

- Automated build (compile, package)
- Automated unit tests
- Automated integration tests
- Automated code quality checks

### 3. Fail Fast and Fail Early

> *"Fail fast and early"* is one of the core mantras of CI.

| Timing of Failure | Cost to Fix | Team Impact |
|-------------------|-------------|-------------|
| Same day the bug is introduced | Very low | Developer remembers the change |
| After 1 week | Medium | Context is fading |
| After 6 months (Big Bang) | Very high | Nobody remembers what caused it |

### 4. Failures Are Normal — But Must Be Addressed

In CI, build failures are expected and healthy. The key rule:

> **A broken build is the team's top priority.** No new work until the build is green.

This enforces a culture of shared ownership over code quality.

### 5. Everyone Integrates Daily

Each developer integrates their work with the main branch at least once a day. This prevents long-lived branches and merge conflicts.

## CI/CD Tooling Overview

CI requires tools to automate the integration pipeline.

### Popular CI/CD Tools

| Tool | Type | Notes |
|------|------|-------|
| **Jenkins** | Open-source, self-hosted | Most widely used; highly extensible via plugins |
| **GitHub Actions** | Cloud-native (GitHub) | Integrated with GitHub repos; YAML-based |
| **GitLab CI/CD** | Cloud + self-hosted | Built into GitLab; powerful pipeline DSL |
| **CircleCI** | Cloud-native | Fast; good Docker/container support |
| **Azure DevOps Pipelines** | Cloud (Microsoft) | Tight Azure integration |
| **AWS CodePipeline** | Cloud (AWS) | Native AWS CI/CD orchestration |
| **ArgoCD** | Kubernetes CD | GitOps-based continuous delivery for K8s |

### What CI Tools Automate

```
Code Push
   │
   ├── 1. Source Code Checkout
   ├── 2. Dependency Installation
   ├── 3. Code Compilation / Build
   ├── 4. Unit Tests
   ├── 5. Code Quality / Lint / SAST ( Static Application Security Testing.)
   ├── 6. Artifact Packaging (JAR / Docker Image)
   ├── 7. Integration Tests
   ├── 8. Artifact Publishing (Nexus / ECR / DockerHub)
   └── 9. Trigger CD pipeline (deploy to staging/prod)
```

### Key Takeaways

- **CI = integrate continuously**, not at the end
- **Automation is mandatory** — CI without automation is not CI
- **Fail fast** — surface bugs immediately so they are cheap to fix
- **Failures in CI are normal**; the goal is to detect and fix them quickly
- Every organisation doing DevOps uses some form of CI pipeline
