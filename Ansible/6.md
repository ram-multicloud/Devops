## Ansible Playbooks

- Ansible playbook is written in a YAML file and is a collection of **plays**.
- Each **Play** is a collection of tasks (and handlers).
- Each **task** represents a unit of activity which is achieved by a **module**.

---

## Ansible Module

- This is the smallest (atomic) unit in Ansible where we can express **desired state**.
- Ansible comes inbuilt with many modules. In addition, we can add community modules or develop our own.

---

## Sample Playbook

```yaml
---
- name: install apache
  become: yes
  hosts: all
  tasks:
    - name: update packages and install apache
      ansible.builtin.apt:
        name: apache2
        update_cache: yes
        state: present
    - name: enable and start apache service
      ansible.builtin.systemd:
        name: apache2
        enabled: yes
        state: started
```

### Equivalent Shell Script

```bash
#!/bin/bash
sudo apt update
sudo apt install apache2 -y
sudo systemctl enable apache2
sudo systemctl start apache2
```

## Basic Playbook Structure

A playbook is a collection of plays:

```yaml
- name: play1
- name: play2
  ...
- name: playn
```

Each play includes:
- **hosts** — where to execute
- **become** — whether to run with sudo
- **tasks** — desired state of individual steps

```yaml
- name: play1
  hosts: <where to run>
  become: yes
  tasks:
    - name: task1
      <module>:
        parameter1: <value1>
        ...
        parametern: <valuen>
        state: <desired state>
```

## WOW (Ways of Working)

1. Find out all the **manual steps** required to configure.
2. Ensure they are working.
3. For each step(s), find an **Ansible module** where you can express the desired state.

---

## Example 1: Install Apache and PHP using Ansible Playbook

### Manual Steps

Create a VM for manual steps and find out commands to install Apache and PHP.

#### 1. Update System Packages

```bash
sudo apt update
```

#### 2. Install Apache Web Server

```bash
sudo apt install apache2 -y
```

Start Apache and enable it to start at boot:

```bash
sudo systemctl start apache2
sudo systemctl enable apache2
```

Check Apache status:

```bash
sudo systemctl status apache2
```

> Verify installation by visiting `http://YOUR_SERVER_IP` — you should see the Apache default welcome page.

#### 3. Install PHP

Ubuntu 24.04 comes with PHP 8.x in its default repositories.

```bash
sudo apt install php libapache2-mod-php -y
```

Install common extensions for web development:

```bash
sudo apt install php-cli php libapache2-mod-php -y
```

Verify PHP installation:

```bash
php -v
```

View loaded PHP modules (optional):

```bash
php -m
```

#### 4. Test Apache + PHP

Create a `test.php` file in `/var/www/html/` to verify PHP is working with Apache:

```bash
echo '<?php phpinfo(); ?>' | sudo tee /var/www/html/test.php
```

Visit `http://YOUR_SERVER_IP/test.php` — if you see the PHP info page, installation is successful.

## Finding Modules

Map each command to its Ansible module:

| Shell Command             | Ansible Module |
|---------------------------|----------------|
| `sudo apt install xyz -y` | `apt`          |
| `sudo dnf install xyz -y` | `dnf`          |
| `sudo systemctl restart xyz` | `systemd`   |

**Tip:** Google `<command> in ansible` to find the right module quickly.

---

## Running the Playbook

After filling in the task section, run the playbook with:

```bash
ansible-playbook -i hosts apache.yaml
```
