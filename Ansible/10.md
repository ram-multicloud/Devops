# Topic: Installing Apache Tomcat 10 on Ubuntu

---

## Manual Steps

### Prerequisites
- Ubuntu 20.04/22.04 server
- `sudo` access

---

### Step 1: Install OpenJDK 17

```bash
sudo apt update
sudo apt install openjdk-17-jdk -y
java -version
```

---

### Step 2: Create Tomcat User

Tomcat should run under a dedicated system user with no login shell.

```bash
sudo useradd -m -d /opt/tomcat -U -s /bin/false tomcat
```

| Flag | Meaning |
|------|---------|
| `-m` | Create home directory |
| `-d /opt/tomcat` | Set home dir to `/opt/tomcat` |
| `-U` | Create a group with the same name as the user |
| `-s /bin/false` | No login shell (security) |

---

### Step 3: Download and Extract Tomcat 10

```bash
cd /tmp
wget https://dlcdn.apache.org/tomcat/tomcat-10/v10.1.52/bin/apache-tomcat-10.1.52.tar.gz

sudo tar -xvf apache-tomcat-10.1.52.tar.gz -C /opt/tomcat --strip-components=1

sudo chown -R tomcat:tomcat /opt/tomcat
```

`--strip-components=1` removes the top-level folder from the archive so files land directly in `/opt/tomcat`.

---

### Step 4: Configure tomcat-users.xml

```bash
sudo nano /opt/tomcat/conf/tomcat-users.xml
```

Replace or add before `</tomcat-users>`:

```xml
<role rolename="manager-gui" />
<user username="manager" password="manager_password" roles="manager-gui" />

<role rolename="admin-gui" />
<user username="admin" password="admin_password" roles="manager-gui,admin-gui" />
```

> **Note:** Static file approach — credentials are hardcoded. A dynamic/template approach (Jinja2) is preferred for production.

---

### Step 5: Create systemd Service (optional but recommended)

```bash
sudo nano /etc/systemd/system/tomcat.service
```

```ini
[Unit]
Description=Apache Tomcat 10
After=network.target

[Service]
Type=forking
User=tomcat
Group=tomcat
Environment="JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64"
Environment="CATALINA_HOME=/opt/tomcat"
ExecStart=/opt/tomcat/bin/startup.sh
ExecStop=/opt/tomcat/bin/shutdown.sh
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable tomcat
sudo systemctl start tomcat
sudo systemctl status tomcat
```

---

### Step 6: Verify

Access Tomcat in browser: `http://<server-ip>:8080`

## Ansible Steps

### Project Structure

```
tomcat_playbook/
├── hosts
├── tomcat.yaml
├── host_vars/
│   └── 172.31.37.28.yaml
└── files/
    └── tomcat-users.xml
```

---

### `hosts`

```ini
[appserver]
172.31.37.28
```

---

### `host_vars/172.31.37.28.yaml`

Variables scoped to the target host:

```yaml
---
java_package_name: openjdk-17-jdk
tomcat_user: tomcat
tomcat_group: tomcat
tomcat_home_dir: /opt/tomcat
tomcat_shell: /bin/false
tomcat_download_url: https://dlcdn.apache.org/tomcat/tomcat-10/v10.1.52/bin/apache-tomcat-10.1.52.tar.gz
```

---

### `files/tomcat-users.xml`

Static file copied to managed node:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<tomcat-users xmlns="http://tomcat.apache.org/xml"
              xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
              xsi:schemaLocation="http://tomcat.apache.org/xml tomcat-users.xsd"
              version="1.0">

  <role rolename="manager-gui" />
  <user username="manager" password="manager_password" roles="manager-gui" />

  <role rolename="admin-gui" />
  <user username="admin" password="admin_password" roles="manager-gui,admin-gui" />

</tomcat-users>
```

---

### `tomcat.yaml` (Full Playbook)

```yaml
---
- name: install tomcat 10
  hosts: appserver
  become: yes
  tasks:

    - name: fail if it is not ubuntu
      ansible.builtin.fail:
        msg: This playbook is for Ubuntu only
      when: ansible_facts['distribution'] != "Ubuntu"

    - name: install java
      ansible.builtin.apt:
        name: "{{ java_package_name }}"
        update_cache: yes
        state: present

    - name: create a tomcat group
      ansible.builtin.group:
        name: "{{ tomcat_group }}"
        state: present

    - name: create a tomcat user
      ansible.builtin.user:
        name: "{{ tomcat_user }}"
        create_home: yes
        group: "{{ tomcat_group }}"
        home: "{{ tomcat_home_dir }}"
        shell: "{{ tomcat_shell }}"

    - name: download and extract tomcat
      ansible.builtin.unarchive:
        src: "{{ tomcat_download_url }}"
        dest: "{{ tomcat_home_dir }}"
        owner: "{{ tomcat_user }}"
        group: "{{ tomcat_group }}"
        remote_src: yes
        extra_opts:
          - --strip-components=1

    - name: copy tomcat-users file
      ansible.builtin.copy:
        src: files/tomcat-users.xml
        dest: "{{ tomcat_home_dir }}/conf/tomcat-users.xml"
        owner: "{{ tomcat_user }}"
        group: "{{ tomcat_group }}"
        mode: '0600'
```

---

### Run the Playbook

```bash
ansible-playbook -i hosts tomcat.yaml
```


---

## Notes & Next Steps

- The `tomcat-users.xml` is currently a **static file** — credentials are hardcoded.
- Next improvement: use a **Jinja2 template** (`ansible.builtin.template`) so credentials can be passed as variables (e.g., from `host_vars` or Ansible Vault).
- Consider adding a `systemd` service task to manage the tomcat process lifecycle.
