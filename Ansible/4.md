## Ansible Setup — Password-Based Authentication

![Ansible Setup Diagram](https://i0.wp.com/directdevops.blog/wp-content/uploads/2026/01/ansible6.png?w=750&ssl=1)

### Overview

- Distribution: **Ubuntu 24.04**
- Two nodes required:
  - **ACN** — Ansible Control Node (where Ansible is installed)
  - **Node-1** — Managed node (target machine)

---

## Step 1 — Prepare the Control Node (ACN)

Ensure Python is installed on the control node:

```bash
python3 --version
```

---

## Step 2 — Install Ansible on Ubuntu 24.04

> Official docs: [Installing Ansible on Ubuntu](https://docs.ansible.com/projects/ansible/latest/installation_guide/installation_distros.html#installing-ansible-on-ubuntu)

```bash
sudo apt update
sudo apt install software-properties-common
sudo add-apt-repository --yes --update ppa:ansible/ansible
sudo apt install ansible -y
ansible --version
```

---

## Step 3 — Create an Inventory File

Create a file called `hosts` containing the IP address of Node-1:

```bash
echo ipaddress > hosts
```

> Replace `98.70.49.154` with the actual IP address of your managed node.

---

## Step 4 — Test Connectivity with an Ad-Hoc Command

Run a `ping` module to verify Ansible can reach Node-1:

```bash
ansible -i hosts -m ping all --ask-pass
```

| Flag | Meaning |
|---|---|
| `-i hosts` | Use the `hosts` file as inventory |
| `-m ping` | Run the built-in `ping` module |
| `all` | Target all hosts in the inventory |
| `--ask-pass` | Prompt for SSH password (password-based auth) |

![Ansible Ping Output](https://i0.wp.com/directdevops.blog/wp-content/uploads/2026/01/ansible5.png?w=750&ssl=1)

---

## Step 5 — Prepare Node-1 (Managed Node)

- Create a node (Node-1) running Ubuntu 24.04
- Ensure Python is installed on Node-1 as well:

```bash
python3 --version
```

> Ansible requires Python to be present on **every managed node**.

---

## Quick Reference — Ansible Ad-Hoc Command Syntax

```
ansible -i <inventory> -m <module> <host-pattern> [options]
```
