# SSH Connectivity & How Ansible Uses SSH

## Lab Setup

| Item | Detail |
|------|--------|
| **Cloud Provider** | Microsoft Azure |
| **OS** | Ubuntu Linux |
| **Nodes** | `node-1`, `node-2` |
| **Goal** | Establish SSH between client and two nodes |

Client (Laptop)
     │

## Password-Based SSH

Connect from your client to a remote machine:

```bash
ssh username@<ip-or-hostname>
# Enter password when prompted
```

Create a new user `myuser` on **both** node-1 and node-2:

```bash
sudo adduser myuser
```

## 3. Key-Based SSH Setup

Key-based auth is more secure and required for automation tools like Ansible.

### Step 1 — Generate SSH Key Pair (on your laptop)

```bash
# Check if keys already exist
ls ~/.ssh

# If not, generate a new key pair
ssh-keygen
# Example: generates temp_ed25519 and temp_ed25519.pub
```

### Step 2 — Copy Public Key to Both Nodes

```bash
ssh-copy-id -i ~/.ssh/temp_ed25519 myuser@<node-1-ip>
ssh-copy-id -i ~/.ssh/temp_ed25519 myuser@<node-2-ip>
```

> This adds your public key to `~/.ssh/authorized_keys` on each node.

### Step 3 — Login Without Password ✅

```bash
ssh -i ~/.ssh/temp_ed25519 myuser@<node-1-ip>
ssh -i ~/.ssh/temp_ed25519 myuser@<node-2-ip>
```

## How Ansible Uses SSH

Ansible uses **SSH + Python** to manage remote nodes.

```
┌─────────────┐      SSH + Python      ┌─────────────┐
│   Ansible   │ ──────────────────────► │   node-1    │
│  (Control)  │                         │  (managed)  │
└─────────────┘                         └─────────────┘
```

### Prerequisites on Nodes

| Requirement | Reason |
|-------------|--------|
| **SSH access** | Ansible connects via SSH |
| **Python installed** | Ansible generates & runs Python scripts remotely |

### Ansible reads from Inventory

The **inventory file** holds:
- IP addresses / hostnames of nodes
- Credential / SSH key info
- Connection options
